Week 6: Engaging with the Public

Learning objectives: Learn to create animations and videos of environmental change, build web applications with Google Earth Engine, and effectively communicate remote sensing science to public audiences.

Translating Science to the Public

Figure: Animation showing a three-day progression of Atlantic hurricanes in September 2017.

Welcome to the Engaging with the Public module of your remote sensing course, where you will learn how to translate your work to the public and engage with the community. Throughout the course, you have made excellent progress, learning about different aspects of remote sensing and working with Google Earth Engine. You have explored creating indices, working with elevation models, performing classifications, and honing your skills in JavaScript programming.

In this module, we will explore how to showcase your scientific products to the public in a manner that is engaging and easy to understand. The general public may not understand spectral indices or machine learning classifiers, but they can immediately grasp the power of satellite imagery when presented effectively.

Why Public Engagement Matters

Remote sensing provides a unique perspective on our planet, but its value increases exponentially when shared with broader audiences. Here's why public engagement is crucial:

  • Decision-Making: Policymakers and stakeholders need accessible visualizations to make informed decisions
  • Education: Students and educators benefit from interactive tools that bring Earth science to life
  • Awareness: The public gains awareness of environmental changes affecting their communities
  • Collaboration: Citizen scientists can contribute to data collection and validation
  • Funding: Clear communication of research impacts helps secure continued support

Earth Engine provides powerful tools to bridge the gap between complex geospatial analysis and public understanding. Two of the most effective methods are:

  1. Animations: Time-series visualizations showing environmental change over time
  2. Web Applications: Interactive tools that allow non-technical users to explore data

Creating Animations and Videos

Animations created by Google Earth Engine are incredibly popular on social media and provide an excellent opportunity to highlight environmental changes. A well-crafted animation can communicate years of environmental data in seconds.

What Makes a Good Animation?

  • Clear Time Period: Display dates prominently so viewers understand the temporal context
  • Consistent Scale: Maintain the same spatial extent and zoom level throughout
  • Appropriate Frame Rate: Not too fast (viewers miss details) or too slow (viewers lose interest)
  • Context: Include geographic references (labels, boundaries) to orient viewers
  • Meaningful Change: Choose phenomena that show dramatic or important changes

Popular Animation Themes

  • Seasonal Changes: Agricultural cycles, snow cover, vegetation phenology
  • Disaster Events: Hurricanes, floods, wildfires, volcanic eruptions
  • Urban Growth: City expansion over decades
  • Water Bodies: Lake level changes, glacier retreat, coastal erosion
  • Vegetation: Deforestation, reforestation, drought impacts

Building Web Applications

Web applications provide non-technical users with easy-to-use interfaces that allow them to interact with the scientific products created through remote sensing. Instead of requiring users to write code, you create intuitive tools with buttons, sliders, and dropdown menus.

Benefits of Earth Engine Apps

  • No Installation Required: Users access apps through a web browser
  • No Coding Required: Users interact through graphical interfaces
  • Powered by Cloud Computing: Earth Engine handles all the processing
  • Shareable: Send a simple URL to anyone
  • Interactive: Users can explore their own areas of interest

Key UI Components

  • ui.Panel: Container for organizing other UI elements
  • ui.Button: Triggers actions when clicked
  • ui.Slider: Allows users to select values from a range
  • ui.Select: Dropdown menu for choosing options
  • ui.DateSlider: Interactive date selection
  • ui.Map: Embedded interactive map
  • ui.Chart: Display graphs and plots
// Example: Simple web app structure
// Create a panel for controls
var panel = ui.Panel({style: {width: '300px'}});

// Add a title
var title = ui.Label('My Remote Sensing App', 
    {fontSize: '24px', fontWeight: 'bold'});
panel.add(title);

// Add a button
var button = ui.Button({
    label: 'Calculate NDVI',
    onClick: function() {
        var ndvi = image.normalizedDifference(['B5', 'B4']);
        Map.addLayer(ndvi, {min: 0, max: 1, palette: ['brown', 'green']}, 'NDVI');
    }
});
panel.add(button);

// Add panel to the map
ui.root.insert(0, panel);

Design Principles for Effective Apps

  1. Simplicity: Don't overwhelm users with too many options
  2. Clear Instructions: Provide guidance on how to use the app
  3. Visual Feedback: Show users when processing is happening
  4. Meaningful Defaults: Pre-set reasonable values so the app works immediately
  5. Error Handling: Gracefully handle user errors with helpful messages

Best Practices for Public Engagement

Communication Strategies

  • Tell a Story: Frame data within narratives that resonate with audiences
  • Use Analogies: Compare abstract concepts to familiar experiences
  • Show Scale: Help people understand the magnitude of changes
  • Avoid Jargon: Use plain language instead of technical terminology
  • Cite Sources: Build trust by showing data provenance

Visual Design Tips

  • Color Choices: Use intuitive color schemes (blue = water, green = vegetation)
  • Contrast: Ensure visualizations are readable on different screens
  • Labels: Clearly label all elements, axes, and legends
  • Consistency: Maintain consistent styling across related visualizations

Accessibility Considerations

  • Use colorblind-friendly palettes
  • Provide text descriptions of visual content
  • Ensure apps work on mobile devices
  • Keep file sizes reasonable for slower internet connections

Measuring Impact

Once you've created animations or apps, consider how to measure their effectiveness:

  • Usage Metrics: Track app visits, animation views, shares
  • User Feedback: Collect comments and suggestions
  • Educational Outcomes: Assess whether users gained understanding
  • Decision Influence: Document cases where your work informed decisions
  • Media Coverage: Track mentions in news, blogs, or social media
