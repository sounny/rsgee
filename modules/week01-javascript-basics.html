Week 1: JavaScript Basics

Learn the fundamental JavaScript concepts needed for Google Earth Engine programming

Introduction

This module introduces the basic JavaScript concepts you'll need to work effectively in Google Earth Engine. Don't worry if you've never programmed before - we'll start from the very beginning and build up your skills gradually.

Variables

In a programming language, variables are used to store data values. In JavaScript, a variable is defined using the var keyword followed by the name of the variable. The code below assigns the text "Gainesville" to the variable named city.

var city = 'Gainesville';

Note: Text strings in the code should be surrounded by quotes. You are free to use either ' (single quotes) or " (double quotes), and they must match at the beginning and end of each string. In your programs, it is advisable to be consistent—use either single quotes or double quotes throughout a given script (the code in this book generally uses single quotes for code). Each statement of your script should typically end with a semicolon, although Earth Engine's code editor does not require it.

If you print the variable city, you will get the value stored in the variable (Gainesville) printed in the Console.

print(city);

When you assign a text value, the variable is automatically assigned the type string. You can also assign numbers to variables and create variables of type number. The following code creates a new variable called population and assigns a number as its value.

var population = 140398;
print(population);

Learn more: W3Schools JavaScript Variables Tutorial

Lists (Arrays)

It is helpful to store multiple values in a single variable. JavaScript provides a data structure called a list (also known as an array) that can hold multiple values. We can create a new list using the square brackets [ ] and add multiple values separated by a comma. Try putting this code into your console and running it.

var cities = ['Gainesville', 'Tampa Bay', 'Orlando', 'Miami'];
print(cities);

If you look at the output in the Console, you will see "List" with an expander arrow (▹) next to it. Clicking on the arrow will expand the list and show you its content. You will notice that along with the four items in the list, there is a number next to each value. This is the index of each item. It allows you to refer to each item in the list using a numeric value that indicates its position in the list.

Important: In JavaScript (and most programming languages), lists are zero-indexed. This means the first item has an index of 0, the second item has an index of 1, and so on.

Objects (Dictionaries)

Objects are another useful data structure in JavaScript. While lists store ordered collections of values, objects store key-value pairs. This makes them perfect for storing related data with descriptive names.

var cityInfo = {
  name: 'Gainesville',
  population: 140398,
  state: 'Florida',
  coordinates: [-82.3248, 29.6516]
};

print(cityInfo);

You can access individual properties of an object using dot notation:

print('City name:', cityInfo.name);
print('Population:', cityInfo.population);

Or using bracket notation:

print('State:', cityInfo['state']);

When to use lists vs. objects:

  • Use lists when you have an ordered collection of similar items (e.g., a list of cities, dates, or numbers)
  • Use objects when you want to store related properties with descriptive names (e.g., information about a single city)

Functions

Functions are reusable blocks of code that perform specific tasks. They help you organize your code and avoid repetition. In JavaScript, you can define a function using the function keyword:

function greet(name) {
  var message = 'Hello, ' + name + '!';
  return message;
}

print(greet('Alice'));  // Output: Hello, Alice!
print(greet('Bob'));    // Output: Hello, Bob!

Functions can take parameters (inputs) and return values (outputs). This makes them incredibly powerful for processing data.

Functions in Earth Engine

In Google Earth Engine, you'll often use functions to process images and image collections. For example:

// Function to calculate NDVI
function calculateNDVI(image) {
  var ndvi = image.normalizedDifference(['B8', 'B4']);
  return ndvi;
}

// Apply the function to an image
var ndviImage = calculateNDVI(myImage);

Comments

Comments are essential for making your code readable and maintainable. They allow you to explain what your code does without affecting how it runs.

// This is a single-line comment

/*
  This is a multi-line comment.
  You can use it to explain more complex code.
  Everything between the opening and closing markers is a comment.
*/

var city = 'Gainesville';  // You can also add comments at the end of lines

Best Practice: Comment your code liberally! Your future self (and your collaborators) will thank you. Good comments explain why you're doing something, not just what you're doing.

Practice Exercises

  1. Create variables for your name, age, and hometown. Print them to the console.
  2. Create a list of your five favorite cities. Print the list and then print just the third city.
  3. Create an object that stores information about yourself (name, major, year, favorite remote sensing application).
  4. Write a function that takes two numbers as parameters and returns their sum.
  5. Add comments to explain each section of your code.
