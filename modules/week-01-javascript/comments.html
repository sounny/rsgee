JavaScript Comments

Why Use Comments?

While writing code, it is helpful to add a bit of text to explain the code or leave a note for yourself. It is a good programming practice to add comments in the code explaining each step.

Comments help you and others understand:

  • What the code is supposed to do
  • Why you made certain decisions
  • How complex sections work
  • What needs to be updated or fixed

Single-Line Comments

In JavaScript, you can prefix any line with two forward slashes // to make it a comment. The text in the comment will be ignored by the interpreter and will not be executed.

// This is a comment!

Example with Code

// Define the study area geometry
var geometry = ee.Geometry.Point([-82.3248, 29.6516]);

// Load Landsat 8 imagery
var image = ee.ImageCollection('LANDSAT/LC08/C02/T1_L2')
  .filterBounds(geometry)  // Filter by location
  .filterDate('2023-01-01', '2023-12-31')  // Filter by date
  .first();  // Get the first image

print(image);  // Display image information

Multi-Line Comments

For longer explanations, you can use multi-line comments with /* to start and */ to end:

/*
This script calculates NDVI for Gainesville, Florida
using Landsat 8 imagery from 2023.
Author: Your Name
Date: January 2024
*/

var geometry = ee.Geometry.Point([-82.3248, 29.6516]);

Keyboard Shortcuts

The Code Editor provides a handy shortcut to quickly comment or uncomment lines:

  • Windows/Linux: Ctrl + /
  • Mac: Cmd + /

You can select multiple lines and press the key combination to make them all comments. Press again to reverse the operation.

Best Practices for Comments

DO: Write Clear, Useful Comments

// Calculate NDVI for vegetation analysis
var ndvi = image.normalizedDifference(['B5', 'B4']);

// Set visualization parameters for green vegetation
var visParams = {min: 0, max: 1, palette: ['red', 'yellow', 'green']};

DON'T: State the Obvious

// Bad example - comment doesn't add value
var x = 5;  // Set x to 5
var y = 10;  // Set y to 10
var sum = x + y;  // Add x and y

DO: Explain WHY, Not Just WHAT

// Use median composite to minimize cloud interference
var composite = collection.median();

// Buffer by 1km to capture surrounding land cover context
var bufferedArea = studyArea.buffer(1000);

Comments for Organization

Use comments to organize longer scripts into sections:

// ========================================
// 1. DEFINE STUDY AREA AND DATES
// ========================================
var geometry = ee.Geometry.Point([-82.3248, 29.6516]);
var startDate = '2023-01-01';
var endDate = '2023-12-31';

// ========================================
// 2. LOAD AND FILTER DATA
// ========================================
var collection = ee.ImageCollection('LANDSAT/LC08/C02/T1_L2')
  .filterBounds(geometry)
  .filterDate(startDate, endDate);

// ========================================
// 3. CALCULATE NDVI
// ========================================
var calculateNDVI = function(image) {
  return image.normalizedDifference(['B5', 'B4']).rename('NDVI');
};

Commenting Out Code for Debugging

When troubleshooting, you can temporarily disable code without deleting it:

var image = collection.first();

// Temporarily disabled while testing

// Testing with NDVI visualization instead
Map.addLayer(ndvi, {min: 0, max: 1, palette: ['red', 'green']}, 'NDVI');

Try It Yourself

Practice adding comments to your code:

  • Add a header comment to your next script with your name and purpose
  • Comment each major section of your code
  • Explain why you chose certain parameters or thresholds
  • Use comments to temporarily disable parts of your code while testing
  • Leave notes for yourself about what still needs to be done

Remember

Good comments make your code:

  • Easier to understand when you return to it later
  • More accessible to collaborators and instructors
  • Simpler to debug and modify
  • More professional and maintainable

Get in the habit of commenting as you write code, not after!
